## Project Layout

```
workspace/
├── apps/                        ← All application source code lives here
│   ├── api-server/              ← Express API  (port 5000)
│   ├── admin/                   ← Admin Panel  (port 3000, path /admin)
│   ├── vendor-app/              ← Vendor App   (port 3001, path /vendor)
│   ├── rider-app/               ← Rider App    (port 3002, path /rider)
│   ├── ajkmart/                 ← Expo mobile app
│   └── mockup-sandbox/          ← UI sandbox   (path /__mockup)
├── lib/                         ← Shared packages (db, auth, i18n, ui …)
├── scripts/
│   └── setup-replit.mjs        ← Auto-generates .env secrets
├── artifacts -> apps            ← Symlink (backward compat for old workflow commands)
├── .env                         ← Generated by setup-replit.mjs (gitignored)
└── package.json                 ← pnpm monorepo root
```

## Workflows  ← Configure EXACTLY these four on every fresh import

| Workflow name      | Command                                                                                             | Port | outputType |
|--------------------|-----------------------------------------------------------------------------------------------------|------|------------|
| `Start application`| `cd apps/api-server && PORT=5000 PORT_FALLBACK_ENABLE=false NODE_ENV=development node ./scripts/start-with-restart.mjs` | 5000 | webview    |
| `Admin Panel`      | `PORT=3000 BASE_PATH=/admin pnpm --filter @workspace/admin run dev`                                 | 3000 | console    |
| `Vendor App`       | `PORT=3001 BASE_PATH=/vendor pnpm --filter @workspace/vendor-app run dev`                           | 3001 | console    |
| `Rider App`        | `PORT=3002 BASE_PATH=/rider pnpm --filter @workspace/rider-app run dev`                             | 3002 | console    |

### How to configure (agent instructions)

```javascript
// Run these four configureWorkflow() calls in the code_execution sandbox:

await configureWorkflow({
  name: "Start application",
  command: "cd apps/api-server && PORT=5000 PORT_FALLBACK_ENABLE=false NODE_ENV=development node ./scripts/start-with-restart.mjs",
  waitForPort: 5000,
  outputType: "webview",
  autoStart: true,
});

await configureWorkflow({
  name: "Admin Panel",
  command: "PORT=3000 BASE_PATH=/admin pnpm --filter @workspace/admin run dev",
  waitForPort: 3000,
  outputType: "console",
  autoStart: true,
});

await configureWorkflow({
  name: "Vendor App",
  command: "PORT=3001 BASE_PATH=/vendor pnpm --filter @workspace/vendor-app run dev",
  waitForPort: 3001,
  outputType: "console",
  autoStart: true,
});

await configureWorkflow({
  name: "Rider App",
  command: "PORT=3002 BASE_PATH=/rider pnpm --filter @workspace/rider-app run dev",
```

## Port Mapping

| Internal Port | Service         | External Path     |
|---------------|-----------------|-------------------|
| 5000          | API Server      | `/` (root)        |
| 3000          | Admin Panel     | `/admin/`         |
| 3001          | Vendor App      | `/vendor/`        |
| 3002          | Rider App       | `/rider/`         |

The API server (port 5000) reverse-proxies `/admin`, `/vendor`, `/rider` to the
respective Vite dev servers. All four must be running to avoid 502 errors.

## Fresh Import Checklist (for agents)

When this project is imported from GitHub to a new Replit, do these steps **in order**:

```
[ ] 1. Run post-merge.sh
        bash scripts/post-merge.sh
        (sets git pull.rebase false, installs deps, generates secrets, pushes DB schema)

[ ] 2. Configure the 4 workflows
        (paste the configureWorkflow() block from the Workflows section above
         into the code_execution sandbox)

[ ] 3. Verify all 4 workflows are running
        const wf = await listWorkflows();
        console.log(wf.map(w => w.name + ': ' + w.state));

[ ] 4. Smoke-test the app
        GET /          → AJKMart hub page (200)
        GET /admin/    → Admin login screen (200)
        GET /vendor/   → Vendor onboarding (200)
        GET /rider/    → Rider onboarding (200)
        GET /api/health → { status: "ok" } (200)
```

## Any-Server Setup (VPS / Codespaces / Local)

```bash
# 1. Clone
git clone https://github.com/ajkmartnet/eid-2
cd eid-2

# 2. Run universal setup
bash scripts/setup.sh

# 3. Copy and fill env
cp .env.example .env
# — Edit .env and fill DATABASE_URL and other required vars

# 4. Generate secrets
node scripts/setup-replit.mjs

# 5. Push schema
pnpm --filter @workspace/db run push-force

# 6. Start services (4 terminals or use pm2/tmux)
PORT=5000 pnpm --filter @workspace/api-server run dev
PORT=3000 BASE_PATH=/admin pnpm --filter @workspace/admin run dev
PORT=3001 BASE_PATH=/vendor pnpm --filter @workspace/vendor-app run dev
PORT=3002 BASE_PATH=/rider pnpm --filter @workspace/rider-app run dev
```

## Common Issues & Fixes

| Error                                    | Fix                                                              |
|------------------------------------------|------------------------------------------------------------------|
| `spawn tsx ENOENT`                       | Run `scripts/post-merge.sh` — deps not installed                |
| `502 Bad Gateway` on /admin /vendor /rider | Start Admin Panel, Vendor App, Rider App workflows            |
| `JWT_SECRET is not set`                  | Run `node scripts/setup-replit.mjs`                             |
| `@esbuild/linux-x64 not found`           | Run `pnpm install --no-frozen-lockfile` from workspace root     |
| DB schema drift warning on startup       | Run `pnpm --filter @workspace/db run push-force`                |
