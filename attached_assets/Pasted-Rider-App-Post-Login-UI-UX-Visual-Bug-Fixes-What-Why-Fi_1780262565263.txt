Rider App Post-Login UI/UX Visual Bug Fixes
What & Why
Fix 22 confirmed visual and UX bugs across all post-login Rider App pages — Wallet, Earnings, Profile, Notifications, Settings, History, and the Bottom Navigation bar. Bugs range from critical light-colored backgrounds showing as white patches inside the dark theme, misleading red colors on non-error values, sub-standard touch targets, and unreadable micro-text. All fixes are UI-only; no API or business logic changes.

Done looks like
No light-colored (purple-50, gray-50, gray-100) backgrounds appear inside dark-theme cards — all replaced with dark equivalents
totalWithdrawn in Wallet is shown in neutral white/muted color instead of red (text-error)
Wallet loyalty transaction icon uses a dark background (bg-purple-500/10) instead of bg-purple-50
Profile Quick Actions "My Orders" card uses dark background (bg-purple-500/10 text-purple-400) instead of bg-purple-50 text-purple-600
Profile Quick Actions "Wallet" card icon is clearly visible (no blend with background)
Notifications empty-state icon container uses a dark background instead of from-gray-50 to-gray-100
Notifications unread card shadow uses a dark-compatible value (no shadow-green-100/50)
Wallet balance hide/show button is at least 40×40px (currently 32×32)
Wallet mini stat labels (text-[9px]) raised to text-[10px] minimum for legibility
Bottom Nav language switcher moved ABOVE the nav bar as a separate thin strip, or given larger touch targets (min-h-[28px] px-3)
Bottom Nav tab labels raised from text-[10px] to text-xs (12px)
Settings back button uses a standard ChevronLeft icon instead of ChevronDown rotate-90
Notifications "View All Notifications" button text replaced with T("viewAllNotifications") i18n key
Notifications stat grid labels raised from text-[9px] to text-[10px]
Onboarding Skip button contrast fixed (color raised from #6B7280 to #A0A0A0)
SELECT dropdowns in Profile form get a visible custom dropdown arrow indicator
EarningsChart non-best-day bars use bg-white/10 instead of bg-border-dark for visibility
Out of scope
Auth pages (covered by Task #1)
Any backend/API changes
Functional/logic changes
Admin or Vendor app
New features
