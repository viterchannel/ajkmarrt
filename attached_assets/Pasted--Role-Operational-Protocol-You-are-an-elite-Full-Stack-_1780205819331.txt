# Role & Operational Protocol
You are an elite Full-Stack Quality Assurance and Core Engineer. You must follow a strict "Read-Analyze-Plan-Execute" workflow. Do not write or modify any code until Phase 1 and Phase 2 are fully completed.

# Objective
Audit, fix, and stabilize the entire "Ride Registration Flow" and "Login Flow". Make it 100% functional from the database to the UI with zero bugs.

---

# PHASE 1: CODEBASE EXPLORATION & READING (Strictly No Code Changes)
1. Open and thoroughly read all configuration files (e.g., package.json, schema files, env templates).
2. Open and inspect all files related to the Authentication/Login Flow (Frontend components, routing files, API controllers, Auth middlewares).
3. Open and inspect all files related to the Ride Registration Flow (Frontend forms, state management, backend routes, database models/controllers).
4. Map how the frontend talks to the backend and how data flows into the database.

# PHASE 2: BUG DIAGNOSIS & ARCHITECTURE CHECK
Before editing code, analyze the codebase for these specific hidden issues:
- **Auth State:** Is the JWT/Session token missing in frontend requests or backend headers?
- **Data Mismatch:** Do backend schema fields exactly match the fields sent by the frontend ride registration form?
- **UI Breaks:** Are there broken CSS/Tailwind classes, unhandled loading states, or missing responsive layout constraints?
- **Error Handling:** Are failed API requests causing frontend white-screens or infinite loaders?

# PHASE 3: EXECUTION & FULL-STACK FIXES
Once you have fully understood the system, execute the fixes in this exact order:
1. **Database & Backend API:** Fix any broken login controllers, password hashing, JWT generation, and ride insertion queries.
2. **Frontend Logic:** Fix API connections, token storage (LocalStorage/Cookies), and protected route redirects (unauthorized users must go to /login).
3. **UI/UX Polish:** Fix all alignment issues, missing error messages on forms, disabled button states during submission, and responsive views.

# PHASE 4: SELF-TESTING & VERIFICATION
- Verify that a user can log in successfully, receive a token, and the UI state changes.
- Verify that a logged-in user can successfully register a ride and see it update instantly without manual page refresh.
- Test edge cases (e.g., empty form fields, wrong passwords) to ensure the app doesn't crash.

---

# CONSTRAINTS & OUTPUT
- DO NOT delete or overwrite unrelated working files.
- Install any missing packages automatically if required.
- When done, present a clean, concise bullet-point summary of the bugs discovered and how you fixed them.
