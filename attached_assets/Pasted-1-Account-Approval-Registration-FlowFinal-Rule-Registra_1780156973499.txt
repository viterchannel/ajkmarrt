1. Account Approval & Registration FlowFinal Rule: Registration karte hi rider ka status mandatory approvalStatus = "pending_review" hoga. Naye riders bina admin review ke koi ride ya delivery accept nahi kar sakte.Logic: Document upload karna sirf ek process hai. Admin backend dashboard par har document (ID Card, License) ko manually check karega aur verification ke baad hi status ko approved mark karega taake fraud aur fake accounts ko roka ja sake.2. Final Validation Order (Ride Eligibility)Rider ko jobs (rides/deliveries) tabhi show hongi aur assign hongi jab woh yeh Teeno (3) Gates clear karega. System database mein har request par yeh query check karega:\(\text{Rider\ Active}=(\text{phoneVerified}==\text{true})\ \textbf{AND}\ (\text{approvalStatus}==\text{"approved"})\ \textbf{AND}\ (\text{walletBalance}\ge \text{admin\_defined\_limit})\)Dynamic Minimum Balance: Yeh amount fix nahi hai. Admin dashboard par ek settings panel hoga jahan se admin jab chahe is limit ko badal sakta hai (e.g., Rs. 200 se Rs. 500). System live database se is value ko har baar fetch karega.User Experience (UX): Agar balance kam hoga, toh rider ko screen par block banner dikhega: "Please top up your wallet with minimum Rs. [Dynamic_Amount] to start receiving rides."3. Email Verification & Financial WithdrawalsFinal Rule: Account registration ke waqt email optional hai, lekin payout/withdrawal ke liye email mandatory hai.Flow: Rider jab pehli baar wallet se "Withdraw" button par click karega, system check karega isEmailVerified. Agar false hai, toh payout block ho jayega aur ek pop-up screen aayegi: "Please add and verify your email to securely process your earnings." OTP validation ke baad hi withdrawal function unlock hoga.4. Social Login (OAuth Provider)Final Rule: Yeh sirf UI placeholder nahi hoga. Phase 1 (MVP) mein sirf Google OAuth actual implement hoga.Reason: Android users ka ratio riders mein sabse zyada hota hai, is se onboarding seamless ho jayegi. Facebook/Apple login ko future updates ke liye hold par rakha jayega.5. Delivery Pickup Confirmation (Offline & Low-Literacy Friendly)Rides aur Deliveries ka mechanism alag hoga. Rides mein rider ka single tap kafi hai, par deliveries mein chori aur ghalti rokne ke liye niche diya gaya strict waterfall model chalega:Standard Flow (Online): Delivery merchant ya customer ke app par ek 4-digit short OTP dikhega (uneducated users ke liye yeh easy hai, unhe sirf dekh kar bolna hai). Rider yeh code apne app mein enter karega.Low Internet / Offline Fallback (App to App Local Code): Agar customer/merchant ke paas bilkul internet nahi hai, toh unka app bina internet ke unke phone ke local time base par ek offline secure code generate karega (jaise Google Authenticator karta hai). Rider woh code daal kar process bypass kar sakta hai.Extreme Fallback (SMS & Admin Call): Agar customer ke paas smartphone hi nahi hai ya phone band hai, toh rider customer se manual signature screen par lega, parcel ki photo click karega, aur backend admin helpline ko call karke request trigger karwayega.
