2. Registration (Simplified)
Step 1: Personal Details
Field	Required	Validation
Full Name	Yes	min 2 chars
Phone Number	Yes	03XXXXXXXXX format
Email	No	valid format if provided
CNIC / ID Card Number	Yes	XXXXX-XXXXXXX-X format
City	Yes	text and  dropdown
Area	Yes	text and  dropdown
Full Address	Yes	text
Step 2: Vehicle Information
Field	Required
Vehicle Type	Yes (Bike / Car / Rickshaw / Van)
Plate Number	Yes (e.g., RWP-1234)
Driving License Number	Yes
Step 3: Password & Terms
Field	Required	Validation
Password	Yes	8+ chars, 1 uppercase, 1 number, 1 symbol
Confirm Password	Yes	must match
Terms & Conditions	Yes	checkbox
On "Register" Button Click:
+------------------------------------------------+
|  1. Auto-fetch browser geolocation (lat/lng)    |
|     ├─ If denied → show manual city/area selector  |
|  2. Send POST /api/auth/register + lat/lng      |
+------------------------------------------------+
                    |
                    v
+------------------------------------------------+
|  Backend Validation:                              |
|  • phone unique?      → NO → 409 "Already exists"
|  • Email unique?      → NO → 409 "Already exists" | |
|  • cnic unique?       → NO → 409 "Already exists" |
|  • password strong?    → NO → 400 "Weak password" |
|  • city/area/address? → NO → 400 "Required"      |
|  • All OK → insert user + rider_profile          |
+------------------------------------------------+
                    |
                    v
+------------------------------------------------+
|  Account Created:                               |
|  • isActive = true                               |
|  • phoneVerified = false                         |
|  • emailVerified = false                         |
|  • documentsApproved = false                     |
|  • approvalStatus = "approved"                   |
|  • kycStatus = "none"                            |
+------------------------------------------------+
                    |
                    v
+------------------------------------------------+
|  Backend returns: access_token + refresh_token   |
|  Frontend stores → auto-login → DASHBOARD         |
+------------------------------------------------+

REMOVED from registration (was in old version):

Document upload (CNIC photos, vehicle photo, license photo)

5. Profile & Verification
Verification Card (added to Profile page)
+------------------------------------------------+
|  VERIFICATION STATUS                             |
+------------------------------------------------+
|  ✓ ID Card Number: Already verified (given at    |
|    registration) → Shows as ✅                      |
+------------------------------------------------+
|  🟡 Phone Verification:                        |
|    • Status: NOT VERIFIED                        |
|    • Button: "Send OTP" → POST /verify/phone/send   |
|    • Enter 6-digit code → POST /verify/phone/confirm |
|    • On success: phoneVerified=true               |
|    • Bonus: ₹20 wallet cash (auto-awarded)       |
+------------------------------------------------+
|  🟡 Email Verification:                        |
|    • Status: NOT VERIFIED (if email provided)    |
|    • Button: "Send OTP" → POST /verify/email/send   |
|    • Enter 6-digit code → POST /verify/email/confirm |
|    • On success: emailVerified=true               |
+------------------------------------------------+
|  🟡 Document Upload (KYC):                     |
|    • Status: NOT SUBMITTED                       |
|    • Button: "Upload Documents"                    |
|    • Upload: CNIC front, CNIC back, License       |
|    • POST /verify/documents (multipart)            |
|    • kycStatus=pending → admin approves           |
|    • On approval: documentsApproved=true           |
+------------------------------------------------+

6. Accept Ride / Active Order
+------------------------------------------------+
|  1. New ride order arrives via Socket.IO         |
|  2. Rider sees order details (pickup, drop, fare) |
|  3. Rider taps "Accept"                          |
|                                                  |
|  +-- Feature Gate Check: accept_ride --+          |
|  |  → requires: documents_approved = true        |
|  |  → If FALSE:                                  |
|  |      403 Response: "Upload documents to        |
|  |      accept rides. Go to Profile > Upload"    |
|  |  → If TRUE:                                    |
|  |      Proceed to accept ride                     |
|  +---------------------------------------+        |
|                                                  |
|  4. Ride status: accepted → pickup → in-progress |
|  5. Ride completes → earnings credited to wallet  |
+------------------------------------------------+

8. Socket Events
Event	Direction	Payload	Action
connect	Out	auth token	Join rider room
new_order	In	order details	Show notification + Accept button
order_accepted	In	order_id	Remove from list (another rider got it)
order_cancelled	In	order_id	Update status
rider_location	Out	{ lat, lng }	Send periodic location updates
disconnect	Out	-	Reconnect on network change
9. Feature Gate System
DB Table: feature_rules
role	feature_name	required_verifications	max_limit_per_day
rider	accept_ride	["documents_approved"]	5 (unverified) / unlimited
customer	withdraw_money	["phone_verified", "documents_approved"]	-
vendor	add_product	["documents_approved"]	-
Middleware Flow
+------------------------------------------------+
|  API Request comes in with feature action       |
|  e.g., POST /orders/accept, POST /withdraw      |
+------------------------------------------------+
                    |
                    v
+------------------------------------------------+
|  featureAccess middleware:                       |
|  1. Look up feature_rules for this role+feature  |
|  2. Check user's verification flags:              |
|     • phoneVerified?                              |
|     • emailVerified?                             |
|     • documentsApproved?                          |
|  3. If any required flag is FALSE:               |
|     → 403 Forbidden with clear message             |
|  4. If all required flags are TRUE:              |
|     → Allow request to proceed                     |
+------------------------------------------------+
