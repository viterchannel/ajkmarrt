2. کیا کوئی Real Bugs ہیں؟ (Real Bugs?)
YES — 5 Critical Issues:

Code
BUG #1: Feature Gate Race Condition
  Location: featureAccess middleware
  Problem: wallet_balance_check is fetched from DB per request
           2 simultaneous accept_ride calls → both can pass gate
           then both withdraw from wallet (double-spend possible)
  Fix: SELECT FOR UPDATE on wallet query during gate check

BUG #2: GPS Offline Queue Not Persisted
  Location: gpsQueue.ts, queueManager.ts
  Problem: If app crashes while offline, pings are lost
           No SQLite/storage fallback mentioned
  Fix: Use capacitor-sqlite instead of in-memory Array

BUG #3: Socket.IO Auth Leak Risk  
  Location: socket.tsx → socket.auth = { token: JWT }
  Detail: Socket.IO handshake logs might expose JWT
           Code has comment "avoids JWT in logs" but implementation might still leak
  Fix: Use socket.auth queryParam instead of payload

BUG #4: Missing Verification Status on App Launch
  Location: Home.tsx → GET /api/riders/me
  Problem: No error handling if GET fails
           Rider can see dashboard without verification flags loaded
           User experience broken if API down
  Fix: Retry logic + skeleton loader

BUG #5: No Rate Limiting on OTP Requests
  Location: /api/verification/phone/send, /email/send
  Problem: Rider can spam OTP requests (brute force)
  Fix: Add rate limiting middleware (5 attempts per hour)
