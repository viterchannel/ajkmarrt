# AJKMart — Rider App → Vendor App Mirror Plan
**Tayyar kiya: 31 May 2026**

> **Sawal:** Kya Rider App ki saari features, configs, aur structure ko Vendor App mein laya ja sakta hai, lekin vendor logic ke mutabiq?
>
> **Jawab: Haan, mumkin hai.** Neeche poori list hai — kya same rahega, kya badlega, aur kya bilkul naya hoga.

---

## Samajhne ki baat pehle

| | Rider App | Vendor App (Mirror ke baad) |
|---|---|---|
| **Role** | Delivery/Ride karne wala | Order receive + store chalane wala |
| **Online/Offline** | Rider toggle karta hai availability | Vendor Store Open/Closed toggle karta hai |
| **Incoming** | New ride/order request aati hai | New customer order aata hai |
| **Active Task** | Ride/delivery chal rahi hai | Order prepare ho raha hai |
| **GPS** | Rider ka location track hota hai | Vendor ki location fixed (store address) |
| **Earnings** | Per ride commission milti hai | Per order payment milti hai |
| **KYC** | License + CNIC required | Business docs + CNIC required |

## SECTION 2 — Authentication & Security Gates

Rider App mein 4 gates hain. Vendor App mein same structure, alag conditions:

| Gate | Rider App | Vendor App Mirror |
|---|---|---|
| **Gate 1: Phone Verification** | Phone OTP required | Same — phone OTP required |
| **Gate 2: ID Card Gate** | CNIC submit karo | CNIC + business license submit karo |
| **Gate 3: KYC Gate** | Admin approve kare tab hi active | Admin approve kare tab hi orders aayein |
| **Gate 4: Approval Gate** | Pending screen if not approved | Same pending screen with vendor message |

**Auth Methods (same rahenge):**
- Phone OTP login
- Email/Password login
- Social OAuth (Google/Facebook)
- Magic Link
- Biometric (FaceID/Fingerprint)
- App Lock (PIN/Biometric)

## SECTION 3 — Real-time & Offline Features

| Feature | Rider App | Vendor App Mirror | Note |
|---|---|---|---|
| **WebSocket** | New order, KYC status, admin chat | New customer order, KYC status, admin chat | Same socket events, different handlers |
| **Heartbeat** | GPS ping frequency adjust hoti hai | Store status ping (open/closed sync) | GPS replace hoga store-status ping se |
| **GPS Queue** | Location batching offline mein | Zaroorat nahi (fixed location) | Remove GPS queue |
| **Offline Queue** | Go-online action queue | Accept-order action queue | Same pattern, alag action type |
| **Dashboard Cache** | Read-only snapshot offline | Read-only order history offline | Same |
| **Network Quality** | Adjust GPS polling | Adjust order polling interval | Same hook, alag use |

## SECTION 9 — Kaam ka Sequence (Step by Step)

Agar yeh kaam karna ho to yeh order follow karna hoga:

```
Step 1:  Rider App ka folder structure copy karo vendor-app mein
Step 2:  Auth layer update karo — role "vendor" karo
Step 3:  GPS/Location code remove karo
Step 4:  Home page: "Go Online" → "Store Open/Close" toggle
Step 5:  Active page: ride panel → order preparation panel
Step 6:  Wallet/Earnings: label changes (ride→order, commission→payout)
Step 7:  Chat: customer-vendor messaging enable karo
Step 8:  Reviews: reply feature add karo
Step 9:  VanDriver.tsx → BulkOrder.tsx banana
Step 10: Platform config vendor-specific flags lagao
Step 11: Theme update karo (vendor colors)
Step 12: Naye hooks add karo (useStoreStatus, useOrderFeed)
Step 13: Feature gates vendor ke liye configure karo
Step 14: Test karo — Auth → Gates → Dashboard → Orders → Wallet
```

## Conclusion

> **Haan, yeh bilkul mumkin hai.** Rider App ka 70% code vendor app mein directly use ho sakta hai. Baqi 30% mein sirf labels, role names, aur vendor-specific logic alag hogi. GPS wali cheezein hata di jayengi aur vendor-specific features (products, store hours, promos) jo pehle se vendor-app mein hain — woh maintain hongi.
>
> **Sabse bada faida:** Dono apps ka UX aur feel ek jaisi hogi — riders aur vendors dono same level ki experience paayenge.
